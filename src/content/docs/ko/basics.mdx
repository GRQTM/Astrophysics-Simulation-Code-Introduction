# 코드 실행 전에 알아야 할 기초

이 페이지는 문서를 열자마자 막히는 초보자를 위한 최소 가이드입니다.
아래 흐름과 용어만 잡아도 설치/실행 문서를 훨씬 덜 헤매고 읽을 수 있습니다.

## 이 페이지를 읽으면 되는 일

- 시뮬레이션 실행이 실제로 무엇을 하는지 이해합니다.
- 첫 실행에서 어떤 설정이 중요한지 우선순위를 잡습니다.
- 초보자가 자주 하는 실수(단위 오류, 너무 큰 첫 실행, 검증 생략)를 피합니다.

## 초보자가 먼저 알아야 할 핵심 개념

- **유체역학 + 중력**: 은하/우주론 문제는 두 물리를 함께 푸는 경우가 많습니다.
- **이산화(Discretization)**: 연속 방정식을 격자 셀이나 입자로 근사합니다.
- **해상도(Resolution)**: 해상도를 올리면 실행시간과 메모리 비용이 빠르게 증가합니다.
- **시간적분(Timestep control)**: 안정성 제한(CFL 계열 등)이 timestep 크기를 결정합니다.
- **초기조건(Initial conditions)**: 컴파일이 정상이어도 초기조건 파일이 틀리면 실행이 실패합니다.
- **경계조건(Boundary conditions)**: 주기경계/고립경계 선택이 결과 거동에 큰 영향을 줍니다.
- **서브그리드 모델(Subgrid)**: 별생성/피드백/냉각은 해상도 아래 물리를 근사한 모델입니다.
- **수렴성 확인(Convergence test)**: 실행 1번으로 결론 내리지 않고 해상도 비교가 필요합니다.

## 첫 실습 전 체크리스트

1. 목표를 한 문장으로 적습니다. 예: "예제를 실행하고 검증 스크립트 통과까지 확인한다." 
2. 처음에는 가장 작은 공식 예제부터 시작합니다.
3. 출력 파일이 예상 폴더에 생성되는지 확인합니다.
4. 크래시 여부만 보지 말고 경고/로그도 같이 읽습니다.
5. 제공된 검증(check) 스크립트가 있으면 반드시 실행합니다.
6. 파라미터는 한 번에 하나씩 바꿔서 재실행합니다.

## 코드 문서를 읽는 효율적인 순서

1. **Getting Started / Running**부터 읽습니다.
2. 실제로 돌릴 example 디렉터리를 먼저 확인합니다.
3. 그 예제에 필요한 config/parameter 항목만 선별해서 읽습니다.
4. 첫 실행 성공 후 방법론 논문을 읽습니다.

이 순서가 초반 과부하를 줄이고 학습 속도를 높여줍니다.

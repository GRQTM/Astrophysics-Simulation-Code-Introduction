## 1. 본실행 환경 확정

- `Config.sh`와 런타임 파라미터 파일을 최종 고정합니다.
- 사용한 소스 코드의 정확한 리비전을 기록하고 버전 관리합니다.
- 본실행 전용 실행 파일을 별도로 빌드합니다.

## 2. 작업 디렉터리 및 스케줄러 구성

```bash
mkdir -p /scratch/<project>/arepo-prod/{ic,output,logs,restart}
cp <ic-file> /scratch/<project>/arepo-prod/ic/
```

SLURM 런처 예시:

```bash
#!/bin/bash
#SBATCH -J arepo_prod
#SBATCH -N 8
#SBATCH --ntasks-per-node=64
#SBATCH -t 24:00:00
#SBATCH -o logs/slurm-%j.out

cd /scratch/<project>/arepo-prod
srun /path/to/Arepo params_prod.txt
```

## 3. 운영 안정성 확보

- 본실행 파라미터로 짧은 파일럿 작업을 먼저 돌려 봅니다.
- 타임스텝 변화, 메모리 사용량, I/O 처리량을 모니터링합니다.
- 긴 작업을 제출하기 전에 재시작 흐름이 정상인지 확인합니다.

## 4. 실행 후 품질 확인

- 전역 보존량과 예상 스케일링 행동을 확인합니다.
- 핵심 진단 지표를 파일럿 및 기준 실행 결과와 비교합니다.
- 실행 파일, 파라미터, 스케줄러 스크립트, 전체 출력을 아카이브합니다.

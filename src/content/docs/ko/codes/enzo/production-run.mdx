## 1. 본실행 입력 고정

- 파라미터 파일과 refinement 설정을 최종 고정합니다.
- 소스 리비전과 컴파일러/MPI 버전을 기록합니다.
- 개발용과 분리된 본실행 실행 파일을 만듭니다.

## 2. 런 디렉토리 준비

```bash
mkdir -p /scratch/<project>/enzo-prod/{input,output,logs,restart}
cp <parameter-files> /scratch/<project>/enzo-prod/input/
```

## 3. 스케줄러 실행

```bash
#!/bin/bash
#SBATCH -J enzo_prod
#SBATCH -N 8
#SBATCH --ntasks-per-node=64
#SBATCH -t 24:00:00
#SBATCH -o logs/slurm-%j.out

cd /scratch/<project>/enzo-prod
srun <path-to-enzo-binary> input/<parameter-file>
```

## 4. 운영 점검

- AMR 레벨 증가, 랭크당 메모리, I/O를 모니터링합니다.
- 체크포인트/재시작을 조기에 검증합니다.
- 주요 출력 블록마다 과학적 sanity check를 수행합니다.

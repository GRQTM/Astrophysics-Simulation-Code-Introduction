## 1. 실행 파일/파라미터 고정

- 컴파일 옵션과 런타임 파라미터를 최종 확정합니다.
- 소스 리비전, 툴체인 버전, 빌드 로그를 보관합니다.
- 캠페인 전체에서 동일한 실행 파일을 사용합니다.

## 2. 작업 경로 준비

```bash
mkdir -p /scratch/<project>/gadget-prod/{ic,output,logs,restart}
cp <ic-files> /scratch/<project>/gadget-prod/ic/
```

## 3. 스케줄러 실행

```bash
#!/bin/bash
#SBATCH -J gadget_prod
#SBATCH -N 8
#SBATCH --ntasks-per-node=64
#SBATCH -t 24:00:00
#SBATCH -o logs/slurm-%j.out

cd /scratch/<project>/gadget-prod
srun /path/to/Gadget4 params_prod.txt
```

## 4. 품질 관리

- 타임스텝 하한, 로드밸런스, 메모리 여유를 모니터링합니다.
- 체크포인트 생성 및 재시작 가능 여부를 점검합니다.
- 핵심 관측량을 파일럿/기준선과 비교합니다.

## 1. 실행 파일 및 설정 고정

- 컴파일 옵션과 파라미터 파일을 최종 고정합니다.
- 소스 리비전과 의존성 버전을 보관합니다.
- 테스트용 바이너리와 본실행 바이너리를 분리합니다.

## 2. 작업 디렉토리 준비

```bash
mkdir -p /scratch/<project>/gizmo-prod/{ic,output,logs,restart}
cp <ic-files> /scratch/<project>/gizmo-prod/ic/
```

## 3. 스케줄러 실행

```bash
#!/bin/bash
#SBATCH -J gizmo_prod
#SBATCH -N 8
#SBATCH --ntasks-per-node=64
#SBATCH -t 24:00:00
#SBATCH -o logs/slurm-%j.out

cd /scratch/<project>/gizmo-prod
srun /path/to/GIZMO params_prod.txt
```

## 4. 운영 점검

- 로드밸런스/타임스텝 적응/메모리 사용량을 모니터링합니다.
- 체크포인트 무결성과 재시작 체인을 확인합니다.
- 실행 중 주기적으로 과학적 sanity check를 수행합니다.

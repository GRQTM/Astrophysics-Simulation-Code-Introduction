# 용어집

이 가이드 전체에서 쓰이는 핵심 용어를 한곳에 정리했습니다. 모든 코드 페이지에서 같은 뜻으로 사용되므로, 코드를 옮겨가며 읽을 때도 혼동이 없습니다.

## 용어 규칙

- 모든 용어의 최종 기준은 **영문 원어**입니다.
- **AMR**, **SPH**, **Hydrodynamics** 같은 핵심 용어는 어떤 코드 문서를 읽든 동일한 의미로 쓰입니다.

## 주요 용어

- **Hydrodynamics (유체역학)**: 시뮬레이션에서 유체의 움직임을 수치적으로 다루는 방법입니다 — 가스가 흐르고, 충돌하고, 충격파를 만드는 과정을 계산합니다.
- **N-body (다체 문제)**: 암흑 물질이나 별과 같은 수많은 입자가 중력으로 서로 끌어당기는 상호작용을 계산하는 방법입니다.
- **AMR (적응형 격자 세분화)**: 계산이 많이 필요한 곳(충격파, 고밀도 영역 등)에만 격자를 촘촘하게 분할해 효율적으로 계산하는 기법입니다.
- **SPH (평활 입자 유체역학)**: 고정된 격자 대신 움직이는 입자들로 유체를 표현하는 방법입니다. 각 입자가 유체 성질을 갖고, 스무딩 커널을 통해 서로 상호작용합니다.
- **Resolution (해상도)**: 시뮬레이션이 구분할 수 있는 가장 작은 공간·질량 규모입니다. 해상도를 높이면 디테일이 늘지만, 계산 비용도 크게 증가합니다.
- **Initial Conditions (초기 조건, IC)**: 시뮬레이션의 출발 상태 — 위치, 속도, 밀도 등입니다. IC가 잘못되면 코드가 아무리 좋아도 결과를 신뢰할 수 없습니다.
